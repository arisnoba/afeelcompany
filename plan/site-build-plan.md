# 기업 사이트 업로드 중심 구축 문서

## Summary

- 사이트는 `심플한 기업 소개 + 갤러리` 구조로 구축한다.
- 핵심은 CMS가 아니라 `내부 관리자용 이미지 업로드 기능`이다.
- 업로드된 이미지는 두 흐름으로 분기한다.
   - 웹사이트 갤러리용
   - 인스타그램 게시용(가능하다면)
- PDF는 파일을 업로드하는 방식이 아니라, 웹사이트 기반 데이터를 이용해 `회사소개서 PDF`를 생성한다.
- 회사소개서 PDF는 사이트의 내용을 기반으로 `소개서 전용 구성(html 템플릿 예상)`을 통해 다운로드 가능하게 만든다.
- (나중에 업데이트) 인스타그램은 초기에는 `반자동 게시`로 운영하고, 게시된 피드는 웹사이트의 별도 갤러리로 노출한다.

## Implementation Changes

### 기술 스택

- 프론트엔드: `Next.js`
- 배포: `Vercel`
- 파일 저장: `Vercel Blob`
- 데이터 저장: `Vercel Postgres` 또는 `Neon`
- 관리자 인증: 내부 관리자 1~2명용 간단 로그인
- 인스타 연동: `Meta Graph API`
- PDF 생성: `Playwright` 기반 서버 PDF 렌더링

### 사이트 구조

- 공개 페이지
   - 홈
   - 회사소개
   - 갤러리
   - 인스타 피드 갤러리
   - Contact
- 관리자 페이지
   - 이미지 업로드
   - 업로드 타입 선택: `웹사이트용`, `인스타용`
   - 포트폴리오(갤러리, 나중엔 셀럽 리스트) 항목 등록/수정/노출 관리
   - PDF 전용 구성 편집(이게 가능한가?)
   - 인스타 게시 대기열 관리 및 수동 게시(나중에 업데이트)

### 데이터 인터페이스

- `GalleryItem`
   - 제목, 설명, 카테고리(남자, 여자, 상의, 하의, 신발, 기타), 이미지, 웹 노출 여부, PDF 노출 여부
- `InstagramQueueItem`
   - 이미지, 캡션, 게시 상태, 게시 시각, 외부 게시 ID
- `CompanyProfileConfig`
   - 회사소개 문구, PDF 전용 섹션 구성, 포트폴리오 포함 순서

### 동작 흐름

- 관리자가 이미지를 업로드한다.
- 업로드 시 콘텐츠 타입을 선택한다.
- `웹사이트용`은 공개 갤러리에 반영한다.
- `인스타용`은 게시 대기열에 저장한다.
- `둘 다`는 두 흐름을 동시에 처리한다.
- PDF 생성 시 `CompanyProfileConfig + GalleryItem`을 조합해 소개서 전용 레이아웃으로 출력한다.

## Test Plan

- 관리자 로그인과 접근 제한이 정상 동작하는지 확인
- 업로드 타입에 따라 웹 갤러리와 인스타 대기열이 올바르게 분기되는지 확인
- 갤러리 이미지가 공개 사이트에 정상 노출되는지 확인
- PDF 전용 구성에서 선택한 항목과 순서가 PDF에 정확히 반영되는지 확인
- 회사소개서 PDF 생성이 레이아웃 깨짐 없이 동작하는지 확인
- 인스타 게시 버튼 실행 시 성공/실패 상태가 기록되는지 확인
- 모바일 화면에서 소개, 갤러리, PDF 관련 흐름이 정상 동작하는지 확인

## Assumptions

- 현재 저장소는 신규 구축 상태다.
- CMS는 사용하지 않는다. shadcn ui를 사용해서 구축한다.
- 업로드 권한은 내부 관리자 1~2명으로 제한한다.
- 인스타그램은 완전 자동이 아니라 검수 후 게시 방식으로 시작한다.
- PDF는 웹사이트 데이터를 재사용하지만 구성은 소개서 전용으로 따로 편집한다.
- 문서 저장 경로 기본값은 `/plan/site-build-plan.md` 로 둔다.
