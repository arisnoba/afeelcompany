  <action>
Create `src/lib/site.ts` as a server-side read helper module with these exact exports and behaviors:

1. `getSiteCompanyProfile(): Promise<SiteCompanyProfile>`
   - query:
     ```sql
     SELECT about_text, contact_email, contact_phone, address
     FROM company_profile
     ORDER BY updated_at DESC
     LIMIT 1
     ```
   - return empty strings when no row exists

2. `getSiteClientBrands(): Promise<SiteClientBrand[]>`
   - query:
     ```sql
     SELECT id, name, logo_url, sort_order
     FROM client_brands
     WHERE is_active = true
     ORDER BY sort_order ASC, created_at ASC
     ```

3. `getFeaturedPortfolio(limit = 6): Promise<PublicPortfolioItem[]>`
   - query `portfolio_items`
   - filter `show_on_web = true`
   - order `sort_order ASC, created_at DESC`
   - clamp output to `limit`

4. `getPublicPortfolioItems(): Promise<PublicPortfolioItem[]>`
   - same query as above without the limit clamp

Implement row mappers inside the file so the pages receive camelCase objects:
- `brand_name -> brandName`
- `celebrity_name -> celebrityName`
- `image_url -> imageUrl`
- `thumbnail_url -> thumbnailUrl`
- `sort_order -> sortOrder`

Import SQL exactly from:

```ts
import { sql } from '@/lib/db'
```

and import types from:

```ts
import type { PublicPortfolioItem, SiteClientBrand, SiteCompanyProfile } from '@/types/site'
```

Do not call the admin API routes from this module. Read Postgres directly.
